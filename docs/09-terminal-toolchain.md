# Fish + Yazi + Micro 终端工具链协同使用教程

本项目在 Debian 系统中部署了 Fish Shell、Yazi 文件管理器和 Micro 编辑器，并对其快捷键进行了对齐与配置共享。本教程介绍这三个工具的独立使用方法、快捷键以及它们在日常运维中的协同工作流。

---

## 1. Fish Shell 命令行交互

Fish 是一个开箱即用的交互式 Shell，具备智能自动建议、语法高亮和简易的脚本语法。

### 1.1 自动建议
Fish 会根据历史输入记录，在键入命令时以灰色虚影实时显示建议：
- **接受完整建议**：按下右方向键 `→` 或 `End` 键。
- **逐词接受建议**：按下 `Alt + →` 或 `Ctrl + F`。
- **不接受建议**：继续输入其他字符，或者按退格键修改。

### 1.2 历史记录检索
- **前缀匹配搜索**：键入命令的开头部分（如 `systemctl`），然后按下 `↑` 或 `↓` 方向键，Fish 将仅在以该前缀开头的历史命令中进行匹配。
- **全局交互式搜索**：按下 `Ctrl + R`，打开交互式历史命令搜索界面，输入关键字进行模糊匹配，按回车执行选中的命令。

### 1.3 目录快捷跳转与缩写
- **快速向上返回**：
  - `..` 相当于 `cd ..`
  - `...` 相当于 `cd ../..`
  - `....` 相当于 `cd ../../..`
- **缩写**：
  在输入缩写并按下空格后，命令行会自动展开为完整的原始命令。
  - 配置方法：使用 `abbr -a 缩写 完整命令`，例如 `abbr -a g git`。
  - 本项目预配置了常用运维缩写，例如输入 `l` 按空格会自动展开为 `ls -la`。

### 1.4 路径同步包装器 `y`
直接运行 `yazi` 退出后，Shell 会保持在启动前的目录。为了实现退出 Yazi 后自动切换到 Yazi 最后的目录，我们在 `~/.config/fish/conf.d/yazi.fish` 中注册了包装器函数 `y`：

```fish
function y
    set tmp (mktemp -t "yazi-cwd.XXXXXX")
    yazi $argv --cwd-file="$tmp"
    if set cwd (cat $tmp); and [ -n "$cwd" ]; and [ "$cwd" != "$PWD" ]
        cd "$cwd"
    end
    rm -f $tmp
end
```
- **使用方法**：在终端中输入 `y` 代替 `yazi` 启动管理器。

## 2. Yazi 终端文件管理器

Yazi 是基于 Rust 开发的非阻塞异步文件管理器。界面分为左（父目录）、中（当前目录）、右（文件或目录预览）三栏。

### 2.1 快捷键指南

#### 2.1.1 目录导航与浏览
| 快捷键 | 对应命令 | 说明 |
| :--- | :--- | :--- |
| `↑` / `↓` | `arrow -1` / `arrow 1` | 光标逐行移动 |
| `PageUp` / `PageDown` | `arrow -100%` / `arrow 100%` | 向上/向下滚动一整屏 |
| `Home` / `End` | `arrow top` / `arrow bot` | 跳转到当前目录的第一个/最后一个项目 |
| `Enter` 或 `→` | `open` / `enter` | 进入文件夹，或以关联程序打开文件 |
| `Backspace` / `Alt+Up` / `←` | `leave` | 返回上一级父目录 |
| `Alt + ←` / `Alt + →` | `back` / `forward` | 回退/前进历史目录（类似于浏览器前后退） |
| `Ctrl + Q` | `quit` | 退出 Yazi 进程 |

#### 2.1.2 项目选择与批量操作
| 快捷键 | 对应命令 | 说明 |
| :--- | :--- | :--- |
| `Space` 或 `Ctrl + Space` | `toggle` | 选中/取消当前文件，光标自动下移 |
| `v` | `visual_mode` | 连续选择模式。按 `v` 开启，移动方向键连续选中，再次按 `v` 退出 |
| `Ctrl + A` | `select_all --state=true` | 全选当前目录下的所有文件 |
| `Esc` | `escape --select` | 清空所有已选状态 |

#### 2.1.3 文件基本管理
| 快捷键 | 对应命令 | 说明 |
| :--- | :--- | :--- |
| `Ctrl + C` | `yank` | 复制选中项目到剪贴板，支持跨目录粘贴 |
| `Ctrl + X` | `yank --cut` | 剪切选中项目 |
| `Ctrl + V` | `paste` | 粘贴刚才复制或剪切的项目 |
| `Delete` | `remove` | 移动到系统回收站 |
| `Shift + Delete` | `remove --permanently` | 彻底物理删除，不经过回收站 |
| `Ctrl + N` | `create` | 新建项目。输入名称以 `/` 结尾创建文件夹，否则创建空白文件 |
| `F2` | `rename` | 重命名文件。默认光标自动定位在后缀名之前 |
| `Ctrl + F` | `filter --smart` | 在当前目录下实时过滤匹配文件名，非全局搜索 |

### 2.2 全局搜索模式
除了当前目录的局部过滤，Yazi 支持在当前目录及所有子目录下进行递归检索：
- **按文件名搜索（`Ctrl + P`）**：
  - 调用 `fd` 进行模糊文件名匹配。
  - 回车直接进入该文件所在的物理目录；按 `Esc` 键退出搜索模式，返回之前的常规视图。
- **按文件内容搜索（`Ctrl + Shift + F`）**：
  - 调用 `ripgrep` 对子目录下所有文件进行文本内容扫描。
  - 输入关键词后，Yazi 虚拟目录会展示包含关键词的文件、对应行号及匹配行内容。
  - 回车调用编辑器直接定位到该行进行修改。
  - 按 `Esc` 键退出搜索模式。

### 2.3 打开方式关联与选择
Yazi 在全局 `~/.config/yazi/yazi.toml` 中通过 MIME 规则和指定程序关联打开方式。
- **默认关联程序**：文本、JSON、JavaScript 和无后缀文件默认关联至 `micro` 编辑器，配置中设置了 `block = true`，使编辑器在前台独占屏幕，退出后返回 Yazi。
- **手动选择打开方式**：选中文件后按下 `Shift + o` 键（即大写字母 `O`），界面会弹出 `Open with` 菜单，输入对应数字编号或用方向键选择，即可使用备用程序（如 `vi`、`nano`）打开文件。

## 3. Micro 终端文本编辑器

Micro 原生支持鼠标操作，快捷键行为与现代图形编辑器一致。

### 3.1 常用快捷键
- **文件与退出**：
  - `Ctrl + S`：保存当前文件。
  - `Ctrl + Q`：退出编辑器。
- **文本编辑**：
  - `Ctrl + C`：复制选中的文本。
  - `Ctrl + X`：剪切选中的文本。
  - `Ctrl + V`：粘贴文本。
  - `Ctrl + Z`：撤销上一步操作。
  - `Ctrl + Y`：重做（反撤销）。
  - `Ctrl + A`：全选当前文件内容。
- **搜索与替换**：
  - `Ctrl + F`：查找文本。在下方输入关键字后，按 `Enter` 或 `Ctrl + N` 查找下一个匹配项，按 `Ctrl + P` 查找上一个。
  - `Ctrl + R`：替换文本。根据提示输入被替换词和替换词。
- **行操作**：
  - `Ctrl + G`：转到指定行。输入行号按回车光标直接跳转。
- **命令行控制**：
  - `Ctrl + E`：打开命令栏，在此可以执行编辑器内部命令（如修改 Tab 大小 `set tabsize 4`）。

### 3.2 多标签与分屏管理
- **多标签操作**：
  - `Ctrl + T`：新建一个空白标签页。
  - `Ctrl + W`：关闭当前标签页。
  - `Alt + ,` / `Alt + .`：向前/向后切换标签页。
- **分屏操作**：
  - 按下 `Ctrl + E` 呼出命令栏。
  - 输入 `split vertical` 进行水平方向切分（左右两屏）。
  - 输入 `split horizontal` 进行垂直方向切分（上下两屏）。
  - **切换分屏焦点**：使用鼠标直接点击对应的分屏，或者按下 `Ctrl + W` 切换光标焦点。

### 3.3 鼠标交互说明
Micro 支持全终端鼠标协议：
- **光标移动**：鼠标左键单选任意字符。
- **文本选择**：按住左键并拖动进行高亮选择；双击选中单词；三击选中整行。
- **滚轮滚动**：支持使用鼠标滚轮上下滚动浏览长文件。

## 4. 工具链协同工作流与场景示例

本节介绍 Fish、Yazi、Micro 三者如何协同配合完成日常运维任务。

### 4.1 场景 A：修改配置并测试服务
* **目标**：修改 `/etc/nginx/nginx.conf` 配置文件并重启服务。
* **步骤**：
  1. 在 Fish Shell 中输入快捷命令 `y` 启动文件管理器。
  2. 导航至 `/etc/nginx/` 目录。
  3. 光标移动到 `nginx.conf`，按 `Enter`。
  4. Yazi 挂起，Micro 启动。找到对应配置行进行修改。
  5. 按 `Ctrl + S` 保存，按 `Ctrl + Q` 退出 Micro。
  6. 界面恢复到 Yazi。按 `Ctrl + Q` 退出 Yazi。
  7. Fish Shell 的当前目录已经自动切换到 `/etc/nginx/`。
  8. 直接执行测试与重启命令：`nginx -t && systemctl restart nginx`。

---

### 4.2 场景 B：全局内容检索与精准定位修改
* **目标**：在多层级源码目录中，查找包含 `install_yazi` 的文件并进行修改。
* **步骤**：
  1. 在 Fish 中输入 `y` 启动 Yazi。
  2. 按下 `Ctrl + Shift + F`。
  3. 输入 `install_yazi` 并回车。
  4. 界面列出匹配的文件及对应行号（例如 `scripts/apps/devops.sh:785`）。
  5. 选中目标行，按 `Enter`。
  6. Micro 被拉起，且光标自动停在第 785 行。
  7. 修改代码，按 `Ctrl+S` 和 `Ctrl+Q` 保存退出。
  8. 屏幕恢复到 Yazi 的搜索结果虚拟目录，按 `Esc` 键退出搜索。

---

### 4.3 场景 C：批量文件操作与 Shell 命令传递
Yazi 允许将选中的文件列表作为参数传递给 Shell 命令，支持以下占位符：
- `%s`：所有已选中文件的绝对路径列表。
- `%d`：当前工作目录的绝对路径。

* **前台阻塞运行示例（按 `:` 键）**：
  1. 在 Yazi 列表中通过 `Space` 键勾选待打包的文件。
  2. 按下冒号键 `:`。
  3. 在提示框中键入：`tar -czvf logs.tar.gz %s`。
  4. 回车，终端清屏并输出压缩进度，执行完后按任意键返回 Yazi。

* **后台非阻塞运行示例（按 `;` 键）**：
  1. 选中要备份的项目，按分号键 `;`。
  2. 在提示框中键入：`cp -r %s /backup/%d/`。
  3. 回车，文件将在后台复制，不影响 Yazi 的界面操作。

---

### 4.4 场景 D：配置共享与多用户继承
为了让所有系统用户（包括 `root`）享受相同的快捷键和工作流，我们采用软链接方式共享真理源用户的配置。
- **配置同步**：
  - Yazi 配置：`~/.config/yazi/yazi.toml` 与 `keymap.toml`
  - Micro 配置：`~/.config/micro/settings.json` 与 `bindings.json`
  - Fish 自动同步脚本：`~/.config/fish/conf.d/yazi.fish`
- **共享实现**：
  部署脚本会自动将其他用户及 `root` 用户的 `~/.config/yazi` 和 `~/.config/micro` 软链接至真理源用户的对应配置目录。所有用户在终端中键入 `y` 均可直接运行配置一致的终端协同工具链。
